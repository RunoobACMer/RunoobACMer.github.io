概率hdu6695

Posted on 2019-08-21

题面

题面链接http://acm.hdu.edu.cn/showproblem.php?pid=6695
有一个小孩，想让他女朋友开心，就去买礼物，有n件礼物，给出每件礼物使女朋友开心的概率，想使对 一个 礼物开心的概率最大，求该概率。保留6位小数。

思路

  • 已知a和b是分别是她开心的概率，选a和b时，对一件礼物开心的概率是(1-a) b + (1 b) a。
  • 要使选两个开心的概率大于选一个的概率，有(1-a) b + (1-b) a > a ，有a>0.5。
  • 所以，对概率排序，若最大的大于0.5，直接输出，因为选任意多个，都会使这个数变小。
  • 若最大的不大于0.5，以下都是猜想：选前k大数，猜想所求概率随着k先变大，在变小。
    （有一个极值点）

经验

大胆假设，不光 敢于假设选前k大数，而且 假设该值先增大后减小。

Gym-101196D图上广搜

Posted on 2019-08-21

题目链接https://vjudge.net/problem/Gym-101196D

题意

要把英语翻译成n种给定语言，m个翻译可以把A语言和B语言相互翻译并有特定花费w。要求翻译成这种语言要与英语的翻译次数最少，此外要保证花费较小。不能翻译成种语言则输出Impossible。否则输出花费。

思路

  1. 用map把语言映射到整型数字
  2. 图上广搜
  3. 同一层vis[]到的数，即使访问过了，也可以更新更小的花费。
  4. 看访问的数是否等于n+1（加上英语），若否，输出Impossible。

细节

  • vis[]不是存0和1，存step。
  • 每次套板子，要仔细看n是否对应，这题的n+1 对应模板的n。

HDU6681 Rikka with Cake

Posted on 2019-08-20

题面

题面链接


题意

给出若干射线（给定坐标和方向），不存在任意两端点有相同的x或者y。求把这个n*m的蛋糕分成几块。

小感慨

一段区间分成两段sort，并不会更慢，还能快一点点。
不过仅仅是一点点，取log之后很小。

树状数组能维护什么

本质上，树状数组能做的事，就是不停的 求和 与 更新 点的值 。
那么为什么本题可以用树状数组呢？
是因为本题可以抽象成在一维坐标轴上的更新和查询。

思路

欧拉公式部分摘自（https://www.cnblogs.com/bpdwn-cnblogs/p/11380757.html）
欧拉公式：V-E+F=2
用人话解释一下：顶点数 减 边数 加 面数 等于 二
V：那么顶点数就是我们的矩形的四个顶点（4） + 射线端点（n） + 射线与矩形交点（n） + 射线与射线的交点（设为c）。
E: 边数是矩形的四个边（4） + 矩形被射线所截而增加的边（n） + 射线的边（n） + 射线被射线所截而增加的边（2*c）
F：所求面数ans + 矩形外平面（1）
由欧拉公式得：V-E+F=2
则有 ：ans = c + 1
即 ans面数等于射线交点加一。

接下来考虑怎么求交点数：

  1. 遍历每一条射线，如向右的射线，找到所有右上方的向下的射线和右下方向上的射线。
    显然超时。
  2. 用树状数组，以向右向上的射线为例。
    (1) 离散化横坐标，1e9存不下。
    (2) 纵坐标从上向下开始找。
    (3) 向右的是add(x,1)，向上的是sum(x)，把sum（x）累计到ans，就是向右向上的射线的交点。
